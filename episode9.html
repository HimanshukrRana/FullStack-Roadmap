2. Design Patterns

TypePatternUse Case
CreationalFactory, Builder, Singleton*Object creation abstraction
StructuralAdapter, Facade, ProxyInterface compatibility, controlled access
BehavioralStrategy, Observer, CommandAlgorithms, event updates, action encapsulation
⚠️ *Singleton: Use cautiously. Prefer dependency injection in modern codebases.

3. HLD Framework

  1. Requirements: Functional (features) + Non-functional (scale, latency, availability)
  2. Estimation: QPS, storage, bandwidth, connections
  3. API Design: REST/GraphQL, endpoints, schemas, pagination
  4. Data Model: SQL vs NoSQL, indexing, sharding strategy

4. Scalability Patterns

graph LR A[Database Scaling] --> B[Read Replicas] A --> C[Sharding: Hash/Range/Directory] D[Caching] --> E[Cache-Aside] D --> F[Write-Through/Behind] D --> G[TTL + LRU Eviction]

Consistency Models: Strong (SQL, transactions) vs Eventual (NoSQL, CRDTs). Choose based on domain.
