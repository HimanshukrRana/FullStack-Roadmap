1. Microservices vs Monolith

FactorMonolithMicroservices
DeploymentSingle unitIndependent services
ScalingVertical/whole appPer-service horizontal
ComplexityLow initial, high laterHigh initial, manageable later
Team StructureFeature teamsDomain teams
⚖️ Rule: Don't start with microservices. Split when monolith becomes bottleneck (team size, deployment frequency, scaling needs).

3. Event-Driven Architecture

// RabbitMQ Publisher
await channel.publish('orders', '', Buffer.from(JSON.stringify({ orderId, userId })));

// RabbitMQ Consumer
channel.consume('order-events', msg => {
  const data = JSON.parse(msg.content);
  processOrder(data);
  channel.ack(msg); // Idempotency key handling required
});
✅ Event Schema: Use CloudEvents standard. Include: id, source, type, time, data. Ensure idempotency with Redis/set.

4. AWS Cloud-Native

• ECR: Container registry
• ECS Fargate: Serverless containers
• Terraform: IaC for VPC, SGs, ALB, Cluster
• VPC Design: Public subnets (ALB), Private subnets (Services), NAT Gateway for outbound

5. Distributed Observability

import { NodeTracerProvider } from '@opentelemetry/sdk-trace-node';
import { BatchSpanProcessor } from '@opentelemetry/sdk-trace-base';
import { OTLPTraceExporter } from '@opentelemetry/exporter-trace-otlp-http';

const provider = new NodeTracerProvider();
provider.addSpanProcessor(new BatchSpanProcessor(new OTLPTraceExporter()));
provider.register();

Three Pillars: Logs (structured), Metrics (Grafana/CloudWatch), Traces (OpenTelemetry).

6. Web3 Fundamentals

graph LR A[Frontend: React + ethers.js] --> B[Wallet: MetaMask] B --> C[Blockchain: Ethereum/Polygon] C --> D[Smart Contract: Solidity]

• Blockchain: Immutable ledger, consensus (PoS)
• Smart Contracts: Self-executing code on-chain
• dApp: Decentralized app (no central server)
• Use Cases: Tokenization, provenance, DAOs, NFTs

⚠️ When NOT to use Web3: Traditional SaaS, high-frequency trading, personal data storage, simple CRUD apps.
